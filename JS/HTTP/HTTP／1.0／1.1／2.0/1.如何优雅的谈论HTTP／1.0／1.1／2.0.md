### HTTP 1.0
* 链接无法复用，即不支持持久链接：
> http 1.0规定浏览器与服务器保持较短的链接。
> 浏览器每次请求都和服务器经常三次握手和满启动。

### HTTP 1.1
* 支持持久链接
语法：
> Connection: keep-alive （HTTP/1.1的请求默认使用一个持久连接）
> Connection: close

* 支持http管道
> 多个http请求可以同时请求，而不用等待相应的响应。服务端响应按照也是按照请求的顺序依次返回（FIFO）。
* 使用多个TCP链接
